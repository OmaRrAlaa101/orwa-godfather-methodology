# Orwa Godfather Methodology

This repository contains personal bug bounty and web application security notes inspired by the Orwa Godfather methodology.

It is organized as a practical recon and exploitation reference covering topics such as:

- Recon and monitoring workflows
- Subdomain enumeration and port scanning
- Fuzzing and endpoint discovery
- Dorking, GitHub/GitLab recon, and leak hunting
- SQL injection, authentication bypass, IDOR, XSS, prototype pollution, and related testing notes

## Main File

- [orwa_methodology.md](orwa_methodology.md) - the core methodology write-up

## How to Use

1. Start with `orwa_methodology.md` for the full workflow.
2. Open the topic-specific files when you want a narrower reference.
3. Keep updating the notes as you refine your own process.

This material is intended for authorized security research, bug bounty work, and defensive learning only. Always test systems only when you have permission.
